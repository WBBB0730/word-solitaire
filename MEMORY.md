- Rewarded ads are routed through `scripts/ads/ad_service.gd`; main gameplay never calls a real ad SDK directly. `scripts/ads/debug_ad_provider.gd` supports the editor default bypass and `ad_bypass` export feature, while `scripts/ads/admob_provider.gd` adapts the real Godot AdMob plugin on Android/iOS.
- Web rewarded ads use `scripts/ads/web_ad_provider.gd` plus the `WordSolitaireWebAds` bridge in `web/shell.html`. The bridge loads Google H5 Ad Placement API in test mode by default, requests `preloadAdBreaks: "on"` before the first `adBreak()`, waits for Google `onReady`, and calls Google-provided `showAdFn()` directly from the rewarded flow so the prop button opens the official mock ad without an extra custom prompt. Each request carries a one-shot `requestId`; Godot ignores callbacks that do not match the current pending request. The JS bridge is frozen after initialization and exposes only request/availability methods. The local test-ad overlay still exists as an explicit fallback path, but it is disabled by default to avoid masking Google mock-ad failures. Production monetization still needs Google H5 Ads approval/client ID and `testMode: false`.
